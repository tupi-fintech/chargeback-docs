# Atualização de Notificação de Ciclo

![Notificações do Sistema](../images/system-notifications.png)

## A Tupi envia uma notificação webhook (HTTP POST) sempre que um ciclo ou etapa de disputa de chargeback for atualizado:

```jsonc
{
    "event": "cycle",
    "payload": {
        "contractDisputeId": "d492bhhnshts7160i5g0", // ID Interno da Tupi
        "new": true, // boolean, true = evento de criação, false = evento de atualização
        "contractDisputeWorkflowStatus": "execution", // enum, aplicar ajustes financeiros somente quando o status for "execution"
        "name": "chargeback", // enum: chargeback | pre_arbitrage | arbitrage
        "status": "open", // enum: open | closed
        "decision": "accepted_in_full", // enum: accepted_in_full | accepted_partially | rejected | null
        "steps": [
            {
                "name": "chargeback", // enum
                "status": "open", // enum: open | closed
                "message": "Chargeback recebido da rede emissora",
                "memberInCharge": "acquirer", // enum: acquirer | issuer | scheme
                "acquirerStepAction": "act", // enum: act | wait | done
                "startDate": "2025-01-01T00:00:00.000Z",
                "endDate": null,
                "actionDueDate": "2025-01-15T00:00:00.000Z"
            }
        ],
        "installmentAdjustments": [
            {
                "installmentNumber": 1,
                "acquirerReferenceNumber": "25699994259255249310221", // ARN
                "issuerDispatchDate": "2025-01-01T00:00:00.000Z",
                "acquirerIssuerFinancialSolution": "full_credit", // enum
                "acquirerIssuerAmount": 10000, // em centavos = 100.00
                "acquirerDispatchDate": null,
                "acquirerLiability": "merchant_debt", // enum
                "acquirerAmount": 10000, // em centavos = 100.00
                "merchantDispatchDate": null,
                "acquirerMerchantFinancialSolution": "debit_merchant", // enum
                "acquirerMerchantAmount": 10000 // em centavos = 100.00
            }
        ],
        "recomendation": "Aceitar o chargeback e reembolsar o portador do cartão",
        "requestForInformation": false,
        "informationReceived": false,
        "startDate": "2025-01-01T00:00:00.000Z",
        "endDate": null
    }
}
```

### Campos da Notificação Webhook

| Campo | Tipo | Obrigatório | Descrição |
|-------|------|-------------|----------|
| `event` | string | Sim | Identificador do tipo de evento. Sempre "cycle" para notificações de atualização de ciclo |
| `payload` | object | Sim | Container para os dados de atualização de ciclo |
| `payload.contractDisputeId` | string | Sim | Identificador único interno da Tupi para o contrato de disputa |
| `payload.new` | boolean | Sim | Indica o tipo do evento. `true` para evento de criação, `false` para evento de atualização |
| `payload.contractDisputeWorkflowStatus` | string | Sim | Status atual do workflow da disputa. Aplicar ajustes financeiros somente quando o status for "execution" |
| `payload.name` | string | Sim | Nome do ciclo da disputa |
| `payload.status` | string | Sim | Status atual do ciclo. Valores: "open", "closed" |
| `payload.decision` | string | Não | Decisão final do ciclo ao ser encerrado. null se ainda estiver aberto |
| `payload.steps` | array | Sim | Array de objetos de etapas representando cada fase dentro do ciclo |
| `payload.installmentAdjustments` | array | Não | Array de objetos de ajuste financeiro por parcela |
| `payload.recomendation` | string | Não | Recomendação da Tupi sobre o resultado da disputa |
| `payload.requestForInformation` | boolean | Sim | Se informações adicionais foram solicitadas às partes envolvidas |
| `payload.informationReceived` | boolean | Sim | Se as informações adicionais solicitadas foram recebidas |
| `payload.startDate` | string | Sim | Data de início do ciclo no formato ISO 8601 |
| `payload.endDate` | string | Não | Data de encerramento do ciclo no formato ISO 8601. Null se ainda não encerrado |

### Campos do Objeto Step

| Campo | Tipo | Obrigatório | Descrição |
|-------|------|-------------|----------|
| `name` | string | Sim | Identificador da etapa dentro do ciclo |
| `status` | string | Sim | Status atual da etapa. Valores: "open", "closed" |
| `message` | string | Não | Mensagem descritiva sobre esta etapa |
| `memberInCharge` | string | Sim | Membro da rede atualmente responsável pela próxima ação |
| `acquirerStepAction` | string | Sim | Ação necessária do adquirente nesta etapa |
| `startDate` | string | Sim | Data de início da etapa no formato ISO 8601 |
| `endDate` | string | Não | Data de encerramento da etapa no formato ISO 8601. Null se não encerrada |
| `actionDueDate` | string | Não | Prazo para a ação necessária no formato ISO 8601 |

### Campos do Objeto Installment Adjustment - `payload.installmentAdjustments[]`

| Campo | Tipo | Obrigatório | Descrição |
|-------|------|-------------|----------|
| `installmentNumber` | number | Sim | Número sequencial da parcela |
| `acquirerReferenceNumber` | string | Sim | Número de Referência do Adquirente (ARN) para esta parcela |
| `issuerDispatchDate` | string | Não | Data em que o emissor despachou o ajuste financeiro no formato ISO 8601 |
| `acquirerIssuerFinancialSolution` | string | Não | Decisão financeira do adquirente na sua relação com o emissor |
| `acquirerIssuerAmount` | number | Não | Valor ajustado no nível do emissor em centavos (ex: 10000 = 100.00) |
| `acquirerDispatchDate` | string | Não | Data em que o adquirente despachou o ajuste financeiro no formato ISO 8601 |
| `acquirerLiability` | string | Não | Classificação de responsabilidade do adquirente pela disputa |
| `acquirerAmount` | number | Não | Valor ajustado no nível do adquirente em centavos (ex: 10000 = 100.00) |
| `merchantDispatchDate` | string | Não | Data em que o ajuste financeiro do estabelecimento foi processado no formato ISO 8601 |
| `acquirerMerchantFinancialSolution` | string | Não | Decisão financeira do adquirente para a parcela do estabelecimento (perspectiva de agenda) |
| `acquirerMerchantAmount` | number | Não | Valor ajustado no nível do estabelecimento em centavos (ex: 10000 = 100.00) |

## Valores de Nome do Ciclo - `payload.name`

| Valor | Descrição |
|-------|-----------|
| `chargeback` | Primeiro ciclo — disputa inicial levantada pelo emissor em nome do portador do cartão |
| `pre_arbitrage` | Segundo ciclo — disputa escalada antes da arbitração formal |
| `arbitrage` | Terceiro ciclo — arbitração formal submetida à bandeira do cartão |

## Valores de Decisão do Ciclo - `payload.decision`

| Valor | Descrição |
|-------|-----------|
| `accepted_in_full` | Valor total da disputa aceito (estabelecimento perde) |
| `accepted_partially` | Valor parcial da disputa aceito |
| `rejected` | Disputa rejeitada — transação devolvida à contraparte |
| `null` | Nenhuma decisão ainda — ciclo aberto |

## Valores de Nome da Etapa - `payload.steps[].name`

| Valor | Descrição |
|-------|-----------|
| `chargeback` | Chargeback inicial recebido da rede emissora |
| `chargeback_response` | Resposta do adquirente ao chargeback inicial |
| `pre_arbitrage` | Escalada de pré-arbitragem levantada pelo emissor |
| `pre_arbitrage_response` | Resposta do adquirente à pré-arbitragem |
| `arbitrage` | Arbitração formal submetida à bandeira do cartão |
| `arbitrage_response` | Resposta do adquirente à arbitração |

## Valores de Membro Responsável - `payload.steps[].memberInCharge`

| Valor | Descrição |
|-------|-----------|
| `acquirer` | O adquirente (cliente da Tupi) é o responsável pela próxima ação |
| `issuer` | O emissor do cartão é o responsável pela próxima ação |
| `scheme` | A bandeira do cartão (Visa, Mastercard, etc.) está processando a etapa |

## Valores de Ação do Adquirente na Etapa - `payload.steps[].acquirerStepAction`

| Valor | Descrição |
|-------|-----------|
| `act` | Ação necessária (responder, fornecer documentação, etc.) |
| `wait` | Aguardando — outra parte é responsável pela etapa atual |
| `done` | Ações concluídas para esta etapa |

## Valores de Acquirer Liability - `payload.installmentAdjustments[].acquirerLiability`

| Valor | Descrição |
|-------|----------|
| `loss` | Adquirente assume o prejuízo. Casos comuns: erro de processamento, falha de autorização |
| `merchant_debt` | Estabelecimento assume o prejuízo. Casos comuns: fraude, desacordo comercial |

## Valores de Acquirer Merchant Financial Solution - `payload.installmentAdjustments[].acquirerMerchantFinancialSolution`

Estes valores representam a decisão financeira do adquirente para um vencimento na sua relação com o estabelecimento.

| Valor | Descrição |
|-------|----------|
| `keep_as_is` | Não fazer nada — manter como está. Se agendada: manter agendada. Se liquidada: não recuperar o dinheiro. Se cancelada: manter cancelada |
| `hold_funds` | Bloquear temporariamente fundos do merchant. Decisão temporária. Se agendada: bloqueia temporariamente o payout (dívida / valor suspeito). Se liquidada: gerar bloqueio temporário de valor equivalente na agenda futura do merchant. Se cancelada: não se aplica |
| `cancel_payout` | O payout será cancelado de forma definitiva. Se agendada: cancelar o payout em definitivo. Se liquidada: não se aplica. Se cancelada: não se aplica |
| `update_amount` | Atualizar o valor em casos de aceite parcial. Se agendada: atualizar o valor do payout. Se liquidada: não se aplica. Se cancelada: não se aplica |
| `debit_merchant` | Prejuízo total pro merchant. Se agendada: não se aplica. Se liquidada: gerar dívida em definitivo do valor equivalente na agenda futura do merchant. Se cancelada: não se aplica |

## Valores de Acquirer Issuer Financial Solution - `payload.installmentAdjustments[].acquirerIssuerFinancialSolution`

Estes valores representam a decisão financeira do adquirente para um vencimento na sua relação com o emissor.

> **Nota:** Os acertos parciais devem ser resolvidos via fee collection.

| Valor | Descrição |
|-------|----------|
| `full_credit` | Crédito para o emissor no valor integral do cupom |
| `accept_chargeback` | Chargeback recebido aceito e débito subsequente permitido |
| `representment` | Chargeback recebido contestado e cobrança mantida. Fluxo geral: enviar uma nova cobrança do cupom dada a discordância com a disputa. Visa Allocation: iniciar uma pré-arbitragem |
| `keep_as_is` | Não fazer nada — manter como está. Se agendada: manter agendada. Se liquidada: não fazer nada. Se cancelada: manter cancelada. Se enviada: não fazer nada. Se confirmada pelo emissor: não fazer nada |
| `cancel_credit` | Cancelar envio de cupons agendados no adquirente. Se agendada: cancelar |

> **Aviso:** O sistema da Tupi não realiza nenhuma decisão ou ajuste financeiro diretamente no sistema do cliente. As notificações descritas nesta documentação têm caráter informativo e refletem as decisões tomadas pelo analista. O recebimento, interpretação e tratamento dos eventos é de exclusiva responsabilidade do sistema do cliente, devendo seguir suas próprias regras de negócio e políticas internas.
