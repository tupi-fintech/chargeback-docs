# Actualización de Notificación de Ciclo

![Notificaciones del Sistema](../images/system-notifications.png)

## Tupi envía una notificación webhook (HTTP POST) cada vez que un ciclo o etapa de disputa de contracargo es actualizado:

```jsonc
{
    "event": "cycle",
    "payload": {
        "contractDisputeId": "d492bhhnshts7160i5g0", // ID Interno de Tupi
        "name": "chargeback", // enum: chargeback | pre_arbitrage | arbitrage
        "status": "open", // enum: open | closed
        "decision": null, // enum: accepted_in_full | accepted_partially | rejected | null
        "steps": [
            {
                "name": "chargeback", // enum
                "status": "open", // enum: open | closed
                "message": "Contracargo recibido de la red emisora",
                "memberInCharge": "acquirer", // enum: acquirer | issuer | scheme
                "acquirerStepAction": "act", // enum: act | wait | done
                "startDate": "2025-01-01T00:00:00.000Z",
                "endDate": null,
                "actionDueDate": "2025-01-15T00:00:00.000Z"
            }
        ],
        "installmentAdjustments": [
            {
                "installmentNumber": 1,
                "acquirerReferenceNumber": "25699994259255249310221", // ARN
                "issuerDispatchDate": "2025-01-01T00:00:00.000Z",
                "acquirerIssuerFinancialSolution": "full_credit", // enum
                "acquirerIssuerAmount": 10000, // en centavos = 100.00
                "acquirerDispatchDate": null,
                "acquirerLiability": "merchant_debt", // enum
                "acquirerAmount": 10000, // en centavos = 100.00
                "merchantDispatchDate": null,
                "acquirerMerchantFinancialSolution": "debit_merchant", // enum
                "acquirerMerchantAmount": 10000 // en centavos = 100.00
            }
        ],
        "recomendation": "Aceptar el contracargo y reembolsar al titular",
        "requestForInformation": false,
        "informationReceived": false,
        "startDate": "2025-01-01T00:00:00.000Z",
        "endDate": null
    }
}
```

### Campos de la Notificación Webhook

| Campo | Tipo | Obligatorio | Descripción |
|-------|------|-------------|----------|
| `event` | string | Sí | Identificador del tipo de evento. Siempre "cycle" para notificaciones de actualización de ciclo |
| `payload` | object | Sí | Contenedor para los datos de actualización de ciclo |
| `payload.contractDisputeId` | string | Sí | Identificador único interno de Tupi para el contrato de disputa |
| `payload.name` | string | Sí | Nombre del ciclo de la disputa |
| `payload.status` | string | Sí | Estado actual del ciclo. Valores: "open", "closed" |
| `payload.decision` | string | No | Decisión final del ciclo al cerrarse. null si todavía está abierto |
| `payload.steps` | array | Sí | Array de objetos de etapas que representan cada fase dentro del ciclo |
| `payload.installmentAdjustments` | array | No | Array de objetos de ajuste financiero por cuota |
| `payload.recomendation` | string | No | Recomendación de Tupi sobre el resultado de la disputa |
| `payload.requestForInformation` | boolean | Sí | Si se solicitó información adicional a las partes involucradas |
| `payload.informationReceived` | boolean | Sí | Si se recibió la información adicional solicitada |
| `payload.startDate` | string | Sí | Fecha de inicio del ciclo en formato ISO 8601 |
| `payload.endDate` | string | No | Fecha de cierre del ciclo en formato ISO 8601. Null si aún no ha cerrado |

### Campos del Objeto Step - `payload.steps[]`

| Campo | Tipo | Obligatorio | Descripción |
|-------|------|-------------|----------|
| `name` | string | Sí | Identificador de la etapa dentro del ciclo |
| `status` | string | Sí | Estado actual de la etapa. Valores: "open", "closed" |
| `message` | string | No | Mensaje descriptivo sobre esta etapa |
| `memberInCharge` | string | Sí | Miembro de la red actualmente responsable de la próxima acción |
| `acquirerStepAction` | string | Sí | Acción requerida del adquirente en esta etapa |
| `startDate` | string | Sí | Fecha de inicio de la etapa en formato ISO 8601 |
| `endDate` | string | No | Fecha de cierre de la etapa en formato ISO 8601. Null si no ha cerrado |
| `actionDueDate` | string | No | Fecha límite para la acción requerida en formato ISO 8601 |

### Campos del Objeto Installment Adjustment - `payload.installmentAdjustments[]`

| Campo | Tipo | Obligatorio | Descripción |
|-------|------|-------------|----------|
| `installmentNumber` | number | Sí | Número secuencial de la cuota |
| `acquirerReferenceNumber` | string | Sí | Número de Referencia del Adquirente (ARN) para esta cuota |
| `issuerDispatchDate` | string | No | Fecha en que el emisor despachó el ajuste financiero en formato ISO 8601 |
| `acquirerIssuerFinancialSolution` | string | No | Decisión financiera del adquirente en su relación con el emisor |
| `acquirerIssuerAmount` | number | No | Monto ajustado en el nivel del emisor en centavos (ej: 10000 = 100.00) |
| `acquirerDispatchDate` | string | No | Fecha en que el adquirente despachó el ajuste financiero en formato ISO 8601 |
| `acquirerLiability` | string | No | Clasificación de responsabilidad del adquirente por la disputa |
| `acquirerAmount` | number | No | Monto ajustado en el nivel del adquirente en centavos (ej: 10000 = 100.00) |
| `merchantDispatchDate` | string | No | Fecha en que el ajuste financiero del comercio fue procesado en formato ISO 8601 |
| `acquirerMerchantFinancialSolution` | string | No | Decisión financiera del adquirente para la cuota del comercio (perspectiva de pagos) |
| `acquirerMerchantAmount` | number | No | Monto ajustado en el nivel del comercio en centavos (ej: 10000 = 100.00) |

## Valores de Nombre del Ciclo - `payload.name`

| Valor | Descripción |
|-------|-----------|
| `chargeback` | Primer ciclo — disputa inicial presentada por el emisor en nombre del titular de la tarjeta |
| `pre_arbitrage` | Segundo ciclo — disputa escalada antes del arbitraje formal |
| `arbitrage` | Tercer ciclo — arbitraje formal presentado a la marca de la tarjeta |

## Valores de Decisión del Ciclo - `payload.decision`

| Valor | Descripción |
|-------|-----------|
| `accepted_in_full` | Monto total de la disputa aceptado (el comercio pierde) |
| `accepted_partially` | Monto parcial de la disputa aceptado |
| `rejected` | Disputa rechazada — transacción devuelta a la contraparte |
| `null` | Sin decisión aún — ciclo abierto |

## Valores de Nombre de la Etapa - `payload.steps[].name`

| Valor | Descripción |
|-------|-----------|
| `chargeback` | Contracargo inicial recibido de la red emisora |
| `chargeback_response` | Respuesta del adquirente al contracargo inicial |
| `pre_arbitrage` | Escalada de pre-arbitraje presentada por el emisor |
| `pre_arbitrage_response` | Respuesta del adquirente al pre-arbitraje |
| `arbitrage` | Arbitraje formal presentado a la marca de la tarjeta |
| `arbitrage_response` | Respuesta del adquirente al arbitraje |

## Valores de Miembro Responsable en la Etapa - `payload.steps[].memberInCharge`

| Valor | Descripción |
|-------|-----------|
| `acquirer` | El adquirente (cliente de Tupi) es responsable de la próxima acción |
| `issuer` | El emisor de la tarjeta es responsable de la próxima acción |
| `scheme` | La marca de la tarjeta (Visa, Mastercard, etc.) está procesando la etapa |

## Valores de Acción del Adquirente en la Etapa - `payload.steps[].acquirerStepAction`

| Valor | Descripción |
|-------|-----------|
| `act` | Acción requerida (responder, proveer documentación, etc.) |
| `wait` | Esperando — otra parte es responsable de la etapa actual |
| `done` | Acciones completadas para esta etapa |

## Valores de Acquirer Liability - `payload.installmentAdjustments[].acquirerLiability`

| Valor | Descripción |
|-------|----------|
| `loss` | El adquirente asume la pérdida. Casos comunes: error de procesamiento, falla de autorización |
| `merchant_debt` | El comercio asume la pérdida. Casos comunes: fraude, desacuerdo comercial |

## Valores de Acquirer Merchant Financial Solution - `payload.installmentAdjustments[].acquirerMerchantFinancialSolution`

Estos valores representan la decisión financiera del adquirente para el cupón en su relación con el comercio.

| Valor | Descripción |
|-------|----------|
| `keep_as_is` | No hacer nada — mantener como está. Si programado: mantener programado. Si liquidado: no recuperar fondos. Si cancelado: mantener cancelado |
| `hold_funds` | Bloquear temporalmente fondos del comercio — retener el dinero con el adquirente. Decisión temporal. Si programado: bloquea temporalmente el pago (deuda / monto sospechoso). Si liquidado: generar bloqueo temporal de monto equivalente en la agenda futura del comercio. Si cancelado: no aplica |
| `cancel_payout` | El pago será cancelado definitivamente. Si programado: cancelar el pago en definitivo. Si liquidado: no aplica. Si cancelado: no aplica |
| `update_amount` | Actualizar monto en caso de aceptación parcial. Si programado: actualizar valor del pago. Si liquidado: no aplica. Si cancelado: no aplica |
| `debit_merchant` | Pérdida total para el comercio. Si programado: no aplica. Si liquidado: generar deuda definitiva del monto equivalente en la agenda futura del comercio. Si cancelado: no aplica |

## Valores de Acquirer Issuer Financial Solution - `payload.installmentAdjustments[].acquirerIssuerFinancialSolution`

Estos valores representan la decisión financiera del adquirente para el cupón en su relación con el emisor.

> **Nota:** Los ajustes parciales deben resolverse mediante fee collection.

| Valor | Descripción |
|-------|----------|
| `full_credit` | Crédito para el emisor por el valor integral del cupón |
| `accept_chargeback` | Contracargo recibido aceptado (débito subsecuente permitido) — no habrá insistencia en el cobro |
| `representment` | Contracargo recibido impugnado y cobro mantenido. Flujo general: enviar un nuevo cobro del cupón dada la discrepancia con la disputa. Visa Allocation: iniciar un pre-arbitraje |
| `keep_as_is` | No hacer nada — mantener como está. Si programado: mantener programado. Si liquidado: no hacer nada. Si cancelado: mantener cancelado. Si enviado: no hacer nada. Si confirmado por el emisor: no hacer nada |
| `cancel_credit` | Cancelar el envío de cupones programados en el adquirente. Si programado: cancelar |

> **Aviso:** El sistema de Tupi no realiza ninguna decisión o ajuste financiero directamente en el sistema del cliente. Las notificaciones descritas en esta documentación tienen carácter informativo y reflejan decisiones tomadas por el analista. La recepción, interpretación y tratamiento de los eventos es de exclusiva responsabilidad del sistema del cliente, debiendo seguir sus propias reglas de negocio y políticas internas.
