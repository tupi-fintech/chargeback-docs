# Status Update Notification

![System Notifications](../images/system-notifications.png)

## Tupi sends a webhook notification (HTTP POST) whenever the status of a chargeback dispute changes:

```jsonc
{
    "event": "status",
    "payload": {
        "contractDisputeId": "d492bhhnshts7160i5g0", // Tupi Internal ID
        "acquirerReferenceNumber": "20099984311312200798013", // ARN (can be null)
        "transactionIdentifier": "431220079801", // TID (can be null)
        "helpdeskCaseIdentifier": "HD-2025-001234", // Helpdesk case ID (can be null)
        "actionStatus": "waiting_for_merchant", // enum
        "workflowStatus": "analysis" // enum
    }
}
```

### Webhook Notification Fields

| Field | Type | Required | Description |
|-------|------|-------------|----------|
| `event` | string | Yes | Event type identifier. Always "status" for status update notifications |
| `payload` | object | Yes | Container for status update data |
| `payload.contractDisputeId` | string | Yes | Tupi's internal unique identifier for the dispute contract |
| `payload.acquirerReferenceNumber` | string | No | Acquirer Reference Number (ARN). Can be null |
| `payload.transactionIdentifier` | string | No | Transaction ID (TID). Can be null |
| `payload.helpdeskCaseIdentifier` | string | No | Helpdesk case identifier associated with this dispute. Can be null |
| `payload.actionStatus` | string | Yes | Current operational status indicating who needs to act on the dispute |
| `payload.workflowStatus` | string | Yes | Current pipeline stage of internal dispute processing |

## Action Status Values - `payload.actionStatus`

The `actionStatus` field indicates the current operational state of the dispute and who is responsible for the next action.

| Value | Description |
|-------|-----------|
| `received_from_issuer_network` | Dispute received from the issuer network |
| `waiting_for_merchant` | Waiting for the merchant to provide a response or documentation |
| `received_from_merchant` | Response received from the merchant |
| `waiting_for_issuer_network` | Response submitted and waiting for the issuer network |
| `done` | Dispute fully resolved |

## Workflow Status Values - `payload.workflowStatus`

The `workflowStatus` field indicates the current stage of the dispute in Tupi's internal processing pipeline.

| Value | Description |
|-------|-----------|
| `draft` | Dispute created but not yet processed |
| `enrichment` | Waiting for transaction or merchant data enrichment |
| `filtering_automation` | Automated filtering rules being evaluated |
| `assignment_automation` | Automated assignment rules being applied |
| `merchant_activation_automation` | Automated merchant activation in progress |
| `review_automation` | Automated review rules being evaluated |
| `assignment` | Dispute being manually assigned to an analyst |
| `analysis` | Dispute under analysis by an analyst |
| `review` | Analyst's work under review |
| `execution` | Dispute response being executed and sent to the network |
| `done` | Workflow completed |
